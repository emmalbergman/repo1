Group One Product Website

Product Info

This is a web application to help manage inventory items. It is a work in progress. Along with basic inventory tracking, the future goals of this project include, but are not limited to:

  • Access control
  • Urgency ranking
  • Notifications when stock runs low
  • Help making reports on inventory data
It will be written using Flask (python) for a web server, Peewee ORM for data storage, and HTML with HTMX for a user interface.

User Documentation

Note: this is a zero release, these features are coming soon!

  • Description

    This is an inventory management system for the HRDC warming center, It allows users to create new products to add to the inventory, update stock, price, image, ect, remove products from inventory, and display the current inventory. Compared to other software like Microsoft Excel, this will be designed specifically for HRDC and be easier to use than more generalized software. This will streamline the process of updating and tracking inventory at the HRDC warming center.

  • Accessing the software

    This is a web application. A user will be able to access it via a url, but for now it can be run locally.

  • Starting the software

    The software can be run by entering the URL into a browser. For a local deployment, refer to the developer documentation.

  • Using the software

    There will be an intuitive user interface that will allow users to create, read, update, and delete entries in the inventory. More complete documentation on using the software will be provided when the application gets further along in development.

    If a bug needs to be reported, a description of the bug can be submitted to Cody Hager (PM): chager004@gmail.com.

Developer Documentation

  • Obtaining the source code

    To get the source code on your machine, run git clone https://github.com/423S25/project1. We intend to implement a package manager, but for now, make sure to have Python 3 installed, with Flask and Peewee.

  • Project Structure

    The directory structure is /docs for the documentation, /src/model for the database code, and /templates for the rendered pages. app.py (the web server) is located in the root directory.
